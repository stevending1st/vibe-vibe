# 2.6.5 家人工具场景：站在他们的角度设计

## 这个场景的特殊挑战

给家人做工具，和给自己做完全不同。

| 给自己做 | 给家人做 |
|---------|---------|
| 你是用户，知道自己要什么 | 你要猜他们需要什么 |
| 你会自己调试、自己修 | 他们遇到问题只会说「用不了」 |
| 你可以接受一定的复杂度 | 他们可能完全不能接受复杂度 |
| 你知道「这个按钮是什么意思」 | 他们可能看不懂界面 |

**核心挑战：你习以为常的东西，对他们可能完全陌生。**


## 思维工具在这个场景的特殊应用

| 思维工具 | 给家人做工具时的特别关注 |
|---------|----------------------|
| JTBD思维 | 他们真正要完成的任务是什么？（往往比你想的简单）|
| 逆向思维 | 什么会让他们用不了？（字太小、步骤太多、需要登录）|
| 减法思维 | 只保留他们真正需要的功能（可能只有1个）|
| 故事思维 | 站在他们的角度想象使用场景：什么时候用？在哪用？|
| 灵魂三问 | 他们是谁？（年龄、技术水平）他们的真实痛点？|


## 用户调研：观察 > 询问

给家人做工具，不能只靠「问」。因为：

- 他们可能说不清自己的需求
- 他们可能说「什么都想要」（因为不知道开发成本）
- 他们可能说「你做的我都喜欢」（因为不想麻烦你）

**更好的方法是观察。**

| 观察点 | 你能发现什么 |
|-------|------------|
| 他们用手机做什么最多 | 微信？看视频？通话？ |
| 他们遇到什么问题会求助你 | 这就是真痛点 |
| 他们怎么用现有的App | 步骤是否顺利？哪里会卡住？ |
| 他们的手机字体设置多大 | 反映他们需要的字号 |


## 非技术用户的设计原则

### 原则1：字要大

| 用户年龄 | 建议基础字号 |
|---------|------------|
| 40-50岁 | 16-18px |
| 50-60岁 | 18-20px |
| 60岁以上 | 20px以上 |

**在 Prompt 里明确说：「字体至少 20px」。**

### 原则2：步骤要少

| 好的设计 | 不好的设计 |
|---------|-----------|
| 打开就能看到信息 | 需要登录才能看 |
| 点一下就完成操作 | 需要选择多个选项 |
| 操作后立刻有反馈 | 操作后没有变化 |

**理想情况：打开→看到→操作→完成，总共不超过3步。**

### 原则3：反馈要明确

| 好的反馈 | 不好的反馈 |
|---------|-----------|
| 「已保存」文字明确出现 | 只是某个图标变了颜色 |
| 成功后有「✓」的动画 | 什么提示都没有 |
| 出错时说「请重试」 | 只显示错误代码 |

### 原则4：容错要高

| 好的设计 | 不好的设计 |
|---------|-----------|
| 点错了可以撤回 | 点错了没有后悔的机会 |
| 填错了有提示「请输入数字」 | 填错了直接报错 |
| 支持多种输入格式 | 格式稍有不对就不认 |

## 实战案例：吃药提醒

### 小明的情况

小明的妈妈62岁，每天要吃3种药（早上、中午、晚上各一次）。妈妈经常忘记吃药，小明想做一个提醒工具。

### 小明的用户调研（观察）

| 观察 | 发现 |
|-----|------|
| 妈妈手机字体设置 | 开到最大 |
| 妈妈最常用的功能 | 微信、打电话 |
| 妈妈怎么记事情 | 用纸条贴在冰箱上 |
| 妈妈之前用过提醒App吗 | 用过，但不会设置，放弃了 |

### 小明给 AI 的 Prompt

```markdown
## 用户是谁
- 我妈，62岁
- 只会用微信，不会安装和设置App
- 视力一般，手机字体开到最大

## 痛点
每天要吃3种药，经常忘。我不在身边，没法每天提醒。

## 使用场景
- 我把网页链接发到家庭微信群
- 妈妈点开链接，看到今天要吃什么药
- 吃完了点一下「已吃」

## 核心功能（只需要这些）
1. 显示今天要吃的药，分早/中/晚三栏
2. 每种药旁边有一个大按钮「已吃」
3. 点了「已吃」后，那一行变成绿色，显示「✓ 已吃」
4. 第二天自动重置（所有药变回「未吃」状态）

## 设计要求
- 字体至少 22px
- 按钮要大，宽度至少占屏幕一半
- 颜色对比要明显（深色字，浅色背景）
- 不需要登录
- 打开就能看到今天的药，不需要任何操作
- 数据保存在浏览器本地

## 不需要的功能
- 不需要添加/编辑药物功能（我帮她在代码里写死）
- 不需要提醒推送（她不会设置权限）
- 不需要历史记录
- 不需要多用户

## 药物列表（直接写在代码里）
- 早上：降压药
- 中午：维生素D
- 晚上：钙片
```

### 这个设计好在哪里

| 设计点 | 为什么这样做 |
|-------|------------|
| 不需要登录 | 减少一个步骤，减少出错可能 |
| 药物写死在代码里 | 妈妈不需要学会「添加药物」 |
| 不需要推送提醒 | 避免权限设置问题，由小明定时发消息提醒 |
| 点击后变绿色+文字提示 | 明确的反馈，妈妈知道操作成功了 |
| 第二天自动重置 | 不需要手动操作 |


## 进阶场景

### 场景1：视频通话快捷入口

```markdown
## 用户
爸妈，60多岁，想和孙子视频但找不到微信视频通话入口。

## 痛点
每次视频都要问「怎么弄」，其实就是微信里点几下，但他们记不住。

## 解决方案
做一个简单网页，上面就一个大按钮「给孙子打视频」。

## 功能
1. 一个大大的按钮，写着「给孙子视频」
2. 点击后弹出提示：
   「1. 打开微信
    2. 找到「小明」的聊天
    3. 点右下角的「+」
    4. 点「视频通话」」
3. 配上截图说明

（其实不能直接调起微信视频，但可以做一个图文教程）
```

### 场景2：生日相册

```markdown
## 用户
妈妈，过生日想给她一个惊喜。

## 功能
1. 打开看到大字「妈妈生日快乐」
2. 下面是家人的祝福语和照片
3. 自动播放背景音乐（可选）
4. 支持左右滑动看不同的祝福

## 设计要求
- 字要大
- 照片要清晰
- 操作要简单（最好只需要滑动）
- 发一个链接就能打开，不需要装任何东西
```

## 核心要点

✓ 给家人做工具，要用「观察」而不只是「询问」来了解需求

✓ 字要大、步骤要少、反馈要明确、容错要高

✓ 不要做「完整」的工具，做「他们能用」的工具

✓ 测试时把自己想象成从没用过智能手机的人
