# 2.6.4 个人工具场景：警惕过度设计的陷阱

## 个人工具的特殊优势

给自己做工具，有一个巨大的优势：**你自己就是用户，不需要猜测需求。**

你知道自己：
- 什么时候会用这个工具
- 在什么设备上用
- 能接受什么程度的复杂度
- 什么功能是真正需要的

这是做产品的人梦寐以求的「用户洞察」——而你天然拥有。


## 个人工具的特殊陷阱

但这个优势也会变成陷阱：**你很容易高估自己的需求。**

| 你想象中的需求 | 实际情况 |
|--------------|---------|
| 我需要分类功能，按工作/生活/学习分开 | 做了之后发现懒得分类，全堆在一起 |
| 我需要统计图表，看看每月花了多少钱 | 做了之后发现从来不看 |
| 我需要多设备同步 | 做了之后发现只在一台设备上用 |
| 我需要提醒功能 | 做了之后发现权限设置太麻烦，放弃了 |

**最后你会发现：做了10个功能，常用的只有2个。**


## 思维工具在个人工具中的应用

| 思维工具 | 在个人工具中要问的问题 |
|---------|---------------------|
| JTBD思维 | 我要完成什么任务？现有的工具为什么不满足我？ |
| 逆向思维 | 什么会让我做完就不用？（太复杂？打开慢？需要填很多东西？） |
| 减法思维 | 第一天必须有的功能是什么？只做这一个 |
| 故事思维 | 我会在什么场景下用？手机还是电脑？白天还是晚上？ |
| 灵魂三问 | 我自己的真实痛点是什么？（不是「可能有用」，而是「真的很痛」） |


## 「过度设计」检测清单

在开始之前，用这个清单检测你的想法是否「过度设计」：

| 信号 | 问题 | 如果答案是「是」 |
|-----|------|----------------|
| 功能太多 | 你的功能列表超过5个了吗？ | 砍到3个以内 |
| 解决假痛点 | 这个痛点你这周遇到过吗？ | 如果没有，可能不是真痛点 |
| 现有工具能用 | 现有的App/工具能解决80%的问题吗？ | 如果能，考虑不做 |
| 想象的使用场景 | 你能具体描述「今天几点我会用它」吗？ | 如果说不清，可能不会用 |
| 完美主义 | 你是不是在想「要做就做好」？ | 先做一个能用的，再说「做好」 |

**如果有3个以上的「是」，建议重新审视这个项目。**


## 个人工具需求描述模板

```markdown
## 我要解决的问题
一句话描述痛点：[如「每天不知道时间花在哪了」]

这个痛点多久出现一次：[每天/每周/偶尔]
出现时我有什么感受：[焦虑/烦躁/沮丧]

## 现有方案为什么不行
我现在怎么解决这个问题：[如「用手机备忘录记」]
为什么不满意：[如「太麻烦，经常忘记记」]

## 使用场景
我会在什么时候用：[如「晚上睡前」]
我会用什么设备：[手机/电脑/都有]
我能接受的操作复杂度：[30秒内完成/1分钟内完成]

## 第一版只需要的功能
只要这一个功能：[如「快速记录今天做了什么」]

不需要的功能（以后再说）：
- [如「分类功能」]
- [如「统计图表」]
- [如「多设备同步」]

## 数据保存
可以保存在浏览器本地（最简单）
```


## 实战案例：极简记账

### 小美的情况

小美每个月发完工资，到月底就不知道钱花哪了。她想做一个记账工具。

### 小美最初的想法

```
我想做一个个人财务管理系统：
- 收支记录
- 分类统计（餐饮/交通/购物/娱乐...）
- 预算管理
- 月度报表
- 图表可视化
- 多账户（微信/支付宝/银行卡）
- 周期性支出（房租/话费）
- 数据导出
```

**问题**：这个列表太长了，做不完。

### 用思维工具重新审视

| 问题 | 小美的诚实回答 |
|-----|---------------|
| 你真正的痛点是什么？ | 月底不知道钱花哪了，想知道「大头」在哪 |
| 这个痛点多久出现一次？ | 每个月底 |
| 你会在什么场景下记账？ | 买完东西用手机顺手记 |
| 你能接受多复杂的操作？ | 最好3秒内完成，太麻烦就懒得记了 |
| 现有App为什么不用？ | 太复杂，每次要选分类，烦 |

### 小美重新定义的第一版

```markdown
## 我要解决的问题
月底不知道钱花哪了，想知道大头在哪。

## 使用场景
- 买完东西在手机上顺手记一下
- 3秒内完成
- 月底看一眼「本月花了多少」

## 第一版只需要
1. 一个输入框，输入金额和简单备注（如「午饭 35」）
2. 一个数字显示「本月累计支出」
3. 能看到最近的记录列表

就这样。不需要分类、不需要图表、不需要多账户。

## 技术要求
- 手机上打开
- 数据保存在浏览器本地
- 不需要登录
```

### 这个版本为什么好

| 对比 | 原版 | 简化版 |
|-----|------|-------|
| 开发时间 | 可能几周 | 可能1小时 |
| 使用门槛 | 需要配置分类、账户 | 打开就能用 |
| 会不会坚持用 | 大概率放弃 | 更可能坚持 |
| 能不能解决核心痛点 | 能，但被其他功能淹没 | 能，而且很直接 |


## 进阶场景

### 场景1：极简番茄钟

```markdown
## 痛点
工作时容易走神，想用番茄钟但现有App太花哨。

## 使用场景
电脑上开着，专注工作时用。

## 第一版只需要
1. 点一下开始25分钟倒计时
2. 时间到了响一声
3. 显示今天完成了几个番茄

不需要：统计报表、自定义时长、白噪音、任务关联。
```

### 场景2：阅读笔记收集器

```markdown
## 痛点
在微信读书上划了很多线，但散落在各本书里，想集中看。

## 使用场景
读完一本书后，把划线内容整理到一起。

## 第一版只需要
1. 一个输入框，粘贴划线内容
2. 能标注是哪本书的
3. 能浏览所有笔记

不需要：分类标签、搜索、导出、分享。
```


## 「做还是不做」决策流程

```
我有一个工具想法
    ↓
这个痛点这周出现过吗？ → 没有 → 可能是假痛点，观察一下
    ↓ 是
现有工具能解决吗？ → 能 → 用现有工具
    ↓ 不能
我能描述具体使用场景吗？ → 不能 → 想法还不成熟，再想想
    ↓ 能
第一版只做1个核心功能 → 开始
```

## 核心要点

✓ 给自己做工具的优势是「你就是用户」，陷阱是「容易高估需求」

✓ 问自己：这个痛点我这周遇到过吗？如果没有，可能不值得做

✓ 第一版只做一个核心功能，能用就行，其他以后再说

✓ 「做完就不用」的工具，不如「简陋但一直在用」的工具
