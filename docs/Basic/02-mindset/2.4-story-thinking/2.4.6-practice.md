# 2.4.6 实战练习：为你的项目应用故事思维

现在，让我们把本节学到的故事思维工具应用到你自己的项目中。

准备好你想做的那个项目想法，跟着下面的练习一步步完成。


## 练习一：写一个用户故事

用「开端→冲突→解决→结局」的结构，为你的项目写一个用户故事。

### 模板

```markdown
## 用户故事：[给你的用户起个名字]

### 开端：用户的日常
[描述用户是谁，他的日常生活/工作是什么样的]
（1-2 句话）

### 冲突：用户的困境
[描述用户遇到了什么问题，为什么这让他困扰]
（2-3 句话）

### 解决：用户发现你的产品
[描述用户如何使用你的产品，解决了什么问题]
（1-2 句话）

### 结局：用户获得的价值
[描述问题解决后，用户的生活/工作有什么改变，他有什么感受]
（1-2 句话）
```

### 示例

```markdown
## 用户故事：小王

### 开端：用户的日常
小王是一个喜欢读书的产品经理，每天通勤时会在手机上看电子书。

### 冲突：用户的困境
他看书时经常划线标记好句子，但这些划线散落在不同的书里。
想回顾时要一本本翻，很麻烦，慢慢就不划了。
他觉得很多好内容「看过就忘」，很可惜。

### 解决：用户发现你的产品
他发现了一个工具，能自动收集所有书的划线，按书籍整理好。

### 结局：用户获得的价值
现在他通勤时经常翻翻之前的划线，觉得「读过的书真正变成了自己的」。
```


## 练习二：画一个三维用户画像

用三维构建法，为你的目标用户画一个立体的画像。

### 模板

```markdown
## 用户画像：[名字]

### 第一维：表面属性
- 年龄/职业/生活状态：
- 技术熟练度：
- 一句话描述这个人：

### 第二维：行为习惯
- 典型使用场景（时间、地点、状态）：
- 现在怎么解决这个问题：
- 试过什么方法，为什么没用：

### 第三维：深层动机
- 表面上他说需要什么：
- 实际上他害怕什么：
- 真正渴望什么感受：
```

### 小技巧：连问 5 次「为什么」

填写第三维时，对你写的每个答案问「为什么」：

| 问题 | 我的回答 |
|-----|---------|
| 用户需要什么？ | [你的回答] |
| 为什么需要这个？ | [继续追问] |
| 为什么这很重要？ | [继续追问] |
| 如果没有会怎样？ | [继续追问] |
| 他最担心的是什么？ | [这通常就是深层动机] |


## 练习三：画一个简化版用户旅程

为你的用户画一个简化版的旅程地图，关注四个关键阶段。

### 模板

```markdown
## 简化版用户旅程

### 阶段 1：发现
- 用户如何知道这个产品/工具：
- 用户的第一反应：
- 这个阶段的关键挑战：

### 阶段 2：首次使用
- 用户第一步会做什么：
- 可能遇到的困惑：
- 期望给用户什么感受：

### 阶段 3：核心流程（日常使用）
- 用户最常重复的操作是什么：
- 这个流程目前顺畅吗（1-5 分）：
- 如何让这个流程更好：

### 阶段 4：长期价值
- 用户为什么会持续使用：
- 什么会让用户推荐给别人：
```

### 常见问题排查

| 问题 | 症状 | 解决方法 |
|-----|------|---------|
| 用户太抽象 | "年轻人""职场人士" | 具体到一个人，给他起个名字 |
| 只有表面属性 | 只写了年龄职业 | 补充行为习惯和深层动机 |
| 痛点不够深 | "想要更方便" | 连问 5 次「为什么」 |
| 故事是自己的 | 把自己当用户 | 问「真正的用户会这样想吗」 |


## 进阶：把练习成果变成 Prompt

完成练习后，你可以用这个模板把成果整合成一个故事化 Prompt：

```markdown
我想做 [你的项目]，先介绍一下背景：

## 我是谁
[从练习二的第一维提取]

## 我的现状
[从练习二的第二维提取]

## 我的痛点
[从练习二的第三维提取]

## 我的期望
[从练习一的「结局」提取]

## 用户旅程
[从练习三提取，可以简化为几个关键点]

请帮我 [设计/实现/规划] 这个项目。
```


## 本节要点

三个练习帮你把故事思维落地到自己的项目：

1. **用户故事**：用四幕结构描述用户从困境到解决的过程
2. **三维画像**：从表面属性、行为习惯、深层动机三个维度理解用户
3. **用户旅程**：关注发现、首次使用、核心流程、长期价值四个阶段

完成这些练习后，你就拥有了一份可以直接用于 Prompt 写作的「用户故事素材库」。
