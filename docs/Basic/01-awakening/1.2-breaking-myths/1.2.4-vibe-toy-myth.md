# 1.2.4 迷信四："Vibe Coding 只是玩具"

> "用 AI 写代码？那不就是玩玩吗，能做正经东西吗？"

对Vibe Coding的实际能力范围存在普遍误解。

实际答案是：**Vibe Coding有明确的边界，但其能力范围远超普遍认知。**


## Vibe Coding 能做什么

先说能做的。这个清单可能会让你惊讶：

### 个人工具类
- ✅ 待办清单、记账本、习惯打卡
- ✅ 个人博客、作品集网站
- ✅ 简历页面、个人主页
- ✅ 学习笔记整理工具

### 工作效率类
- ✅ 数据可视化看板
- ✅ 表单收集工具
- ✅ 简单的自动化脚本
- ✅ 内部使用的小工具

### 创意项目类
- ✅ 互动网页、小游戏
- ✅ 活动邀请函、电子贺卡
- ✅ 产品原型、概念验证
- ✅ 数据分析报告

### 学习项目类
- ✅ 课程作业、毕业设计原型
- ✅ 技术概念验证
- ✅ 个人技能展示

**这些，Vibe Coding 都能搞定。**


## Vibe Coding 不适合什么

但也有一些场景，纯 Vibe Coding 确实不太够：

| 场景 | 为什么不适合 |
|------|-------------|
| 涉及资金交易 | 安全性要求高，需要专业审计 |
| 处理敏感数据 | 隐私合规要求，不能靠"差不多" |
| 高并发系统 | 性能优化需要专业知识 |
| 长期维护的产品 | 代码质量和可维护性很重要 |
| 对外发布的商业产品 | 需要更严谨的开发流程 |

**这些场景需要专业开发者介入**，或者你需要进阶学习更多知识。


## 正确的定位

把 Vibe Coding 想象成**瑞士军刀**：

- 它不是专业的厨师刀，切不了最复杂的菜
- 但它能帮你解决 80% 的日常需求
- 而且随身携带，随时能用

**Vibe Coding 的价值在于：**

1. **快速验证想法**：几小时做出原型，而不是几周
2. **降低入门门槛**：不需要先学半年再动手
3. **自己解决小问题**：不用等技术部门排期
4. **学习的起点**：从做出东西开始，再深入学习


## 真实数据

来看看一些有意思的数据：

**Y Combinator 2025 年的统计：**
> 25% 的创业公司，代码的 95% 是 AI 生成的

这些不是玩具项目，而是拿到投资、有用户的真实产品。

**GitHub 2024 年的调查：**
> 使用 AI 编程工具的开发者，编码速度提升 55%

连专业开发者都在用 AI 辅助。

**关键不在于"用不用 AI"，而在于"用得好不好"。**


## 从"玩具"到"工具"的进阶路径

如果你担心 Vibe Coding 只是玩玩：

### 阶段 1：纯 Vibe
- 用自然语言描述需求
- AI 生成代码
- 看效果、调整、迭代

**你能做出**：个人工具、简单网页

### 阶段 2：Vibe + 代码阅读
- 开始看懂 AI 生成的代码
- 理解大致的结构和逻辑

**你能做出**：更复杂的应用、开始修改细节

### 阶段 3：Vibe + 手动修改
- 能自己改一些小地方
- 知道什么时候 AI 出错了

**你能做出**：质量更高的项目、能解决更多问题

### 阶段 4：AI 辅助开发
- AI 是你的助手，不是唯一依赖
- 有独立的技术判断能力

**你能做出**：商业级产品、专业项目

**本教程会带你从阶段 1 走到阶段 2。** 进阶版会带你走得更远。


## 小结

| 迷信 | 真相 |
|-----|------|
| Vibe Coding 只能做玩具 | 能做很多实用的东西 |
| AI 生成的代码不靠谱 | 用对了场景，完全够用 |
| 专业开发不需要 AI | 连专业开发者都在用 |

**Vibe Coding 是工具，不是玩具。**

**关键在于你用它来做什么。**


::: warning 重要提醒
Vibe Coding 有边界，这很正常。

任何工具都有适用范围。

知道它能做什么、不能做什么，比盲目吹捧或贬低都重要。

这本书会帮你找到这个边界，并教你如何在边界内把事情做好。
:::
