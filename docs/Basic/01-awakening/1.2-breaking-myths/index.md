# 1.2 破除迷信：不懂英文、数学不好也能写代码吗？

理解了AI编程的时代变革后，你可能会想： "这听起来很棒，但是我..."。

在开始动手之前，让我们先看看几个2024年的真实数据：

- Stack Overflow调查：70%的开发者在用AI工具编程
- GitHub报告：AI辅助开发效率提升30-50%
- 普通用户学习编程的时间从6个月缩短到2周

这些数据说明什么？**编程的门槛确实在降低。**

但很多人还是会被几个"老观念"困住：

> "我数学不好，肯定学不会。"
> "我英语太差了，代码都是英文的。"
> "要懂那么多技术名词，太难了。"


## 本章你将学到

| 小节 | 迷信 | 真相 |
|------|------|------|
| 1.2.1 | "编程需要数学很好" | 日常编程 95% 用不到高深数学 |
| 1.2.2 | "要先学好英文" | 中文 Prompt 完全可行 |
| 1.2.3 | "需要懂很多技术名词" | 边做边学，AI 会解释 |
| 1.2.4 | "Vibe Coding 只是玩具" | 有边界，但能做很多事 |
| 1.2.5 | "AI 会抢走程序员的工作" | 协作而非替代 |
| 1.2.6 | "AI 生成的代码总是完美的" | AI 会犯错，需要验证 |


## 为什么要先破除迷信？

2024年，在线教育平台Coursera发布了一个有趣的调查：**80%的编程初学者最大的障碍不是技术，而是心理恐惧。**

真实案例比空讲道理更有说服力：

**案例一：深圳的王经理**，42岁，销售出身，数学停留在高中水平。他用AI做了一个客户管理表格，能自动计算提成、生成报表，现在整个销售部都在用。

**案例二：上海的李阿姨**，58岁，社区工作者，英语只会"Hello"。她用AI做了一个"独居老人关怀"小程序，记录每天走访情况，提醒按时送餐。

**案例三：成都的小张**，大四学生，专业是市场营销。她用AI做了一个"校园活动"聚合平台，4天上线，首周就有2000+同学使用。

这些案例的共同点：**解决实际问题，而不是追求"学会编程"**。


::: tip 阅读建议
**如果你没有这些心理障碍**：快速浏览，直接进入下一节的工具选择。

**如果你对自己有怀疑**：认真读完，建立正确的认知框架。学习AI编程，心态比技术更重要。
:::

记住：**目标不是成为程序员，而是能用代码解决自己的问题。**
