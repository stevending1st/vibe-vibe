# 1.2.3 迷信三："需要懂很多技术名词"

> "什么 API、框架、后端、数据库……听都听不懂，怎么学？"

面对大量技术术语，初学者感到困惑是正常的。

关键在于：**你不需要先掌握这些术语，再开始动手实践。**


## 传统学习 vs AI 时代学习

**传统方式：**
```
学概念 → 学语法 → 学框架 → 做项目
（可能需要半年到一年）
```

**AI 时代的方式：**
```
说出你想要的 → 做出来 → 遇到问题再学
（可能只需要一下午）
```

**顺序完全反过来了。**

你不需要先搞懂什么是"前端"、"后端"、"数据库"，才能做一个网站。

你只需要说："帮我做一个能记录待办事项的网页"。

AI 会帮你处理所有技术细节。


## 常见技术术语解释

为便于理解，以下是几个核心术语的简要说明：

| 技术名词 | 大白话解释 |
|---------|-----------|
| 前端 | 用户能看到的部分（页面、按钮、颜色） |
| 后端 | 用户看不到的部分（数据处理、存储） |
| API | 两个程序之间的"传话筒" |
| 数据库 | 存数据的地方，像个超级 Excel |
| 框架 | 别人写好的代码模板，让你不用从零开始 |
| 部署 | 把你的网站放到网上，让别人能访问 |

**知道这些就够了。** 用到的时候再深入了解。


## 边做边学的正确姿势

与其先啃概念，不如：

### Step 1：先动手做
不管懂不懂，先让 AI 帮你生成代码，看看效果。

### Step 2：遇到问题，问 AI
```
AI 说要配置一个 API，这是什么意思？
能用更简单的话解释一下吗？
```

### Step 3：记住"够用就好"
不需要完全理解每个概念。知道它是干什么的，能用就行。

### Step 4：逐渐积累
做的项目多了，这些词自然就熟悉了。


## 实际案例

**某小学教师，45岁，语文专业背景**

目标：制作一个学生作业提交管理网页。

初始状态：完全不了解前端、后端等概念。

直接向AI提出需求：

```
制作一个网页，支持学生姓名输入和作业文件上传，
教师端可以查看提交记录。
```

AI生成完整代码并成功运行。

**三个月后的学习成果**：
- 前端：学生交互界面
- 后端：文件处理逻辑
- 数据库：提交记录存储

关键洞察：概念理解发生在项目完成之后，而非学习开始之前。

实践验证：**先做出成果，再理解概念**的学习方式更为高效。


## 一个小提醒

虽然不需要先学概念，但有一个能力很重要：

**愿意搜索和提问。**

遇到不懂的词，你可以：
- 直接问 AI："XX 是什么意思？"
- 用搜索引擎查一下
- 在本教程的附录术语表里找

**好奇心比知识储备更重要。**


## 小结

| 迷信 | 真相 |
|-----|------|
| 要先学会技术名词 | 边做边学，遇到再查 |
| 不懂概念就做不了 | AI 会帮你处理技术细节 |
| 学编程要从基础学起 | 先做出东西，再补基础 |

**避免让知识缺口阻碍实践行动。**

这种转变大幅降低了学习门槛，提高了学习效率。

**关键原则**：好奇心比现有知识储备更重要。
