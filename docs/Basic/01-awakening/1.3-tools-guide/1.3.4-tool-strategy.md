# 1.3.4 本教程的工具策略

你可能在想：「这个教程到底用什么工具？我要不要跟着装一样的？」

这一节，我们来聊聊本教程的"工具哲学"。


## 我们的策略：不绑定任何工具

**本教程不会要求你必须用某个特定工具。**

为什么？三个原因：

### 原因 1：工具变化太快

2024 年最火的工具，2025 年可能就被新工具超越了。

- Cursor 曾经一家独大，现在 Windsurf 在追赶
- Bolt.new 去年才出现，今年已经是零基础首选
- 明年又会有新的工具出来

如果我们绑定某个工具，这份教程很快就会过时。

### 原因 2：方法论比工具重要

我们教的是**思维方式**，不是**按钮在哪**。

当你学会了：
- 如何把想法拆解成 AI 能理解的描述
- 如何迭代式地改进
- 如何 Debug（修复问题）

这些能力在任何工具上都能用。工具只是载体，方法论才是核心。

### 原因 3：每个人情况不同

- 有人能装软件，有人只能用网页
- 有人在国内，有人在国外
- 有人愿意付费，有人想先用免费的

强制大家用同一个工具，反而会给很多人造成障碍。


## 本教程的使用方式

### 我们会怎么写示例

在教程中，当我们需要演示操作时：

1. **给出通用的 Prompt**（提示词），而不是具体的按钮操作
2. **解释背后的思路**，而不是死记硬背的步骤
3. **偶尔用某个工具截图**，但会说明"其他工具操作类似"

```
✅ 我们会这样写：
「把这段描述发给 AI：帮我做一个计时器，能开始、暂停、重置」

❌ 我们不会这样写：
「打开 Cursor，点击左上角的 Chat，输入...」
```

### 你应该怎么跟着学

1. **选一个你能用的工具**（参考上一节的推荐）
2. **把教程里的 Prompt 复制过去**
3. **看效果，对比教程的预期结果**
4. **如果不一样，问 AI 帮你调整**

不同工具的界面不同，但核心流程是一样的：**描述需求 → 看结果 → 迭代改进**。


## 本节小结

- **本教程不绑定特定工具**，教的是通用的方法论
- **工具选择**：参考 [1.3.3](./1.3.3-choose-tools.md) 选类别，[1.3.1](./1.3.1-tool-overview.md) 选具体工具
- **方法论比工具重要**：学会思路，换工具只是换界面
