## 摘要

前 Google 首席决策科学家 Cassie Kozyrkov 在这篇文章中，试图给出一个兼顾数据、风险与实践的 vibe coding 定义，核心关键词包括 vibe coding、decision intelligence、risk management、prompt engineering、AI safety。 她一方面承认“用意图和感觉驱动代码生成”在提升试验速度上的巨大潜力，另一方面强调如果在缺乏度量与审计的情况下贸然上生产，就好比在不知道损失分布的前提下不断下注，是典型的反模式。 创新点在于她提出“你不一定想做 vibe coding，但你一定要理解别人正在这么做”的立场：文章从决策思维出发，建议团队为 vibe coding 建立清晰的风险分层，从玩具实验、内部工具到用户面对的关键系统，各自需要不同级别的验证和治理。 文中虽没有给出详尽数据，但通过风险矩阵、决策树等概念，将 vibe coding 放进组织决策与责任框架之中，帮助读者从“好不好用”转向“在什么场景、承担多大风险使用”的更成熟视角。
