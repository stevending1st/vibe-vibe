## 摘要

作者通过完整的 ScubaDuck 项目案例研究，论证 vibe coding 能够生产出可用的软件系统，项目共计 150 个提示，关键词包括 vibe coding case study、ScubaDuck、JavaScript UI、prompt-driven development、LLM-generated tests。 文中详细记录从项目启动到功能完善的每一轮对话迭代，强调虽然无法"一次性完成"，但通过让 LLM 同时生成测试代码，开发者能够在修复 bug 时避免回归问题，从而保持代码演化的稳定性。 案例的价值在于提供了真实存在证明（existence proof），表明 vibe coding 在 JavaScript UI 领域是可行的开发策略，同时作者也谨慎指出这并非适用所有领域，读者需要结合具体场景建立对何时有效、何时无效的直觉，为实践者提供了务实的参考标准。​
