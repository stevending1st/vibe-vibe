## 摘要

这场 YC 演讲中，Karpathy 从“软件工程三次范式变迁”谈起，将 vibe coding 放入从手写代码、到云与微服务、再到大模型与智能体的历史脉络中，核心关键词包括 vibe coding、agentic coding、prompt-driven development、software 2.0。 他强调未来大量开发工作会从“写函数”转向“对系统讲故事”：用 PRD、约束和示例对 LLM 进行高密度上下文喂养，让模型以自动化 agent 形式负责代码生成、重构和集成，开发者更多成为 orchestrator 与产品架构师。 创新点在于把 vibe coding 从“玩票式对话写代码”提升为工程工作流光谱的一端：从简单 prompt 实验到完整 agent pipeline，并提出“结构化 prompt、长期记忆、规则与工具接入”四个要素来约束 AI，以避免完全凭直觉乱改代码。 演讲还引用早期 Copilot 采用率、自动化测试覆盖率等行业数据，提醒团队在追求速度的同时必须建立测试和安全防护网，否则 vibe coding 很容易演变为大规模技术债生成器。
