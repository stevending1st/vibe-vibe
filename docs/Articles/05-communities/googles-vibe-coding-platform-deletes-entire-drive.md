## 摘要

The Register 报道一款与 Google 相关的 vibe coding / AI 开发平台在一次严重事故中误删用户整块磁盘，这一事件迅速在 Reddit 等社区引发“AI 工具是否可靠”的大讨论，核心关键词包括 vibe coding tools behaving badly、data loss、AI-driven development risks、safety incident。 报道引用多名用户在 Reddit 的求助与吐槽贴，描述他们如何在“让 AI 自动清理项目目录”的操作中遭遇整个驱动器被擦除的灾难，并将此归因于模糊指令、权限设计不当以及缺乏明显的风险提示。 争议点在于：到底是用户对 vibe coding 工具的误用，还是产品在权限和 UX 设计上的重大缺陷？不少开发者在讨论中指出，如果 AI 助手可以执行 rm -rf 级别的敏感操作，那么必须引入多重确认、沙盒和审计日志等安全机制，而不能假设用户都懂底层风险。 虽然文章本身未给出具体数字，但相关安全厂商随后在博客与 webinar 中引用这一事件，呼吁为 vibe coding 建立类似 DevSecOps 的治理与审计框架，使其真正适配生产环境
