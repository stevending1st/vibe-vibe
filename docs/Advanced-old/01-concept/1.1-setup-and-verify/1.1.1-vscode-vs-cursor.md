# 1.1.1 神器如何选——编辑器选择：VSCode vs Cursor 功能对比

### 一句话破题

Cursor 是 VS Code 的 AI 增强版——它在 VS Code 的基础上深度集成了 AI 能力，让你可以用自然语言和代码"对话"。

### 核心价值

选对编辑器，决定了你后续的开发体验。在 Vibe Coding 时代，编辑器不仅是写代码的地方，更是你与 AI 协作的"指挥部"。

### 快速对比

| 特性 | VS Code | Cursor |
|------|---------|--------|
| **基础编辑** | ✅ 完善 | ✅ 完善（继承 VS Code） |
| **插件生态** | ✅ 海量 | ✅ 兼容 VS Code 插件 |
| **AI 对话** | ⚠️ 需安装插件 | ✅ 原生支持 |
| **代码补全** | ⚠️ 需 Copilot 订阅 | ✅ 内置 AI 补全 |
| **上下文理解** | ⚠️ 有限 | ✅ 可索引整个代码库 |
| **价格** | 免费 | 免费版 + Pro 订阅 |

### 我的建议

**如果你是 Vibe Coding 的初学者，直接选 Cursor。**

原因很简单：Cursor 把 AI 能力做成了"一等公民"。你不需要折腾插件配置，开箱即用。它的核心功能包括：

1. **Cmd/Ctrl + K**：选中代码后，用自然语言描述你想要的修改
2. **Cmd/Ctrl + L**：打开 AI 对话面板，讨论代码问题
3. **Tab 补全**：AI 预测你接下来要写的代码

### 快速上手

1. 访问 [cursor.com](https://cursor.com) 下载安装
2. 首次启动时，它会询问是否导入 VS Code 配置，选择"是"
3. 登录账号，免费版每月有一定的 AI 调用额度

### 避坑指南

- **别纠结于工具选择**：无论选哪个，都能完成本课程。工具是为了解决问题，不是用来比较的
- **VS Code 老用户**：如果你已经有深度定制的 VS Code 配置，可以先用 GitHub Copilot 插件体验 AI 编程，之后再考虑迁移
- **网络问题**：Cursor 需要联网调用 AI 服务，确保你的网络环境畅通
