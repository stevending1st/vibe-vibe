# 5.6.5 我一个人搞得定吗——单人自评

### 一句话破题

独立开发者需要诚实地评估：**这个项目我能在预期时间内完成吗？**

### 可行性自检清单

在开始之前，问自己这些问题：

| 问题 | 是 | 否 |
|------|----|----|
| 我理解业务需求吗？ | 继续 | 需要澄清 |
| 技术栈我熟悉吗？ | 继续 | 需要学习时间 |
| 有类似项目经验吗？ | 更准确 | 估算要加 buffer |
| 依赖的服务都了解吗？ | 继续 | 需要调研 |
| 时间充足吗？ | 继续 | 需要缩减范围 |

### 工作量估算方法

#### 方法一：功能分解法

把项目拆成最小的功能单元，分别估算：

```markdown
## 博客系统工作量估算

### 后端开发
| 模块 | 功能点 | 预估时间 |
|------|--------|----------|
| 用户认证 | 登录/注册/登出 | 4h |
| 文章管理 | CRUD + 发布状态 | 6h |
| 分类标签 | CRUD | 3h |
| 文件上传 | 图片上传 | 3h |

### 前端开发
| 模块 | 功能点 | 预估时间 |
|------|--------|----------|
| 首页 | 文章列表 | 4h |
| 详情页 | 文章渲染 | 3h |
| 后台 | 文章编辑器 | 6h |
| 认证 | 登录表单 | 2h |

### 其他
| 项目 | 预估时间 |
|------|----------|
| 项目初始化 | 2h |
| 部署配置 | 3h |
| 测试调试 | 8h |
| 文档 | 2h |

### 合计
- 开发时间：46h
- 加 30% buffer：60h
- 按每天 4h 有效时间：15 天
```

#### 方法二：类比法

参考类似项目的实际时间：

```
上次做类似项目：10 天
这次更复杂 × 1.5：15 天
新技术学习 + 2 天：17 天
```

### 独立开发者时间陷阱

| 常见低估点 | 实际耗时 |
|------------|----------|
| "这个很简单" | 往往不简单 |
| "查个文档就行" | 可能查半天 |
| "调试应该很快" | 经常最耗时 |
| "部署几分钟" | 环境问题多 |

**经验法则**：初步估算后乘以 1.5 ~ 2

### 有效工作时间

独立开发者一天的真实时间分布：

```
一天 8 小时
├── 有效编码时间：4h（最理想）
├── 调试/查资料：2h
├── 休息/中断：1h
└── 沟通/杂事：1h
```

所以估算时用**有效时间**而不是总时间。

### AI 辅助开发的时间调整

使用 AI 辅助后，某些任务会变快：

| 任务类型 | 传统耗时 | AI 辅助后 |
|----------|----------|-----------|
| 写样板代码 | 1h | 0.2h |
| 查文档/示例 | 0.5h | 0.1h |
| 调试错误 | 1h | 0.5h |
| 学习新 API | 2h | 0.5h |

但要注意：
- 审查 AI 代码需要时间
- 复杂逻辑仍需自己思考
- 集成和测试不会变快

### 让 AI 帮你估算

```
你：帮我估算这个博客系统的开发时间

技术栈：Next.js + Prisma + PostgreSQL
功能：文章 CRUD、分类标签、用户认证、Markdown 编辑
我对 Next.js 熟悉，Prisma 第一次用

AI：## 工作量估算

### 开发时间
| 阶段 | 时间 | 说明 |
|------|------|------|
| 环境搭建 | 2h | 项目初始化、数据库 |
| 数据模型 | 3h | Prisma Schema 设计 |
| 用户认证 | 4h | next-auth 集成 |
| 文章模块 | 8h | CRUD + 编辑器 |
| 分类标签 | 4h | 基础 CRUD |
| 前端页面 | 10h | 列表/详情/后台 |
| 测试调试 | 6h | 各模块联调 |
| 部署上线 | 3h | Vercel + 数据库 |

### 合计
- 基础时间：40h
- Prisma 学习 buffer：+8h  
- 总计：48h
- 按每天 4h：12 天
- 建议预留：2 周
```

### 何时该缩减范围

如果估算结果超出预期，优先考虑：

1. **砍功能**：先做核心功能
2. **用现成方案**：评论用第三方
3. **降低标准**：先能用，再好看
4. **分阶段**：MVP → V1 → V2

### 实用建议

1. **诚实面对自己**：不要自欺欺人
2. **记录实际时间**：下次估算更准
3. **预留 buffer**：总会有意外
4. **及时调整范围**：发现不对劲就缩减
