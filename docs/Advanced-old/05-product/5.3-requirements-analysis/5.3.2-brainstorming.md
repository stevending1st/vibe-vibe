# 5.3.2 我该做什么功能——AI 辅助头脑风暴

### 一句话破题

让 AI 帮你**从一个想法生成十种可能**，然后你来选最好的。

### 头脑风暴的困境

一个人想方案时，容易陷入思维定式：

```
你的想法："做一个博客"
你的方案：文章列表 + 详情页 + 编辑器

问题：这是你见过的博客的样子，但不一定是最好的方案
```

### AI 如何帮你发散思维

**Prompt 模板**：

```markdown
我想做 [产品描述]。

请帮我进行头脑风暴：
1. 列出 5 种不同的产品形态
2. 每种形态说明：核心特点、适合场景、实现难度
3. 推荐最适合独立开发者的方案
```

**示例对话**：

```
你：我想做一个个人博客系统

AI：以下是 5 种不同的博客形态：

## 1. 传统博客
- 核心特点：文章列表 + 分类标签 + 评论
- 适合场景：内容创作者、技术分享
- 实现难度：⭐⭐（中等）

## 2. 知识库/Wiki 风格
- 核心特点：双向链接、知识图谱、全文搜索
- 适合场景：知识管理、学习笔记
- 实现难度：⭐⭐⭐（较难）

## 3. 微博客/说说
- 核心特点：短内容、时间线、快速发布
- 适合场景：碎片化记录、日常分享
- 实现难度：⭐（简单）

## 4. Newsletter 订阅制
- 核心特点：邮件订阅、定期推送、付费墙
- 适合场景：内容变现、精品内容
- 实现难度：⭐⭐⭐（较难）

## 5. 静态博客 + 评论服务
- 核心特点：静态生成、CDN 加速、第三方评论
- 适合场景：技术博客、追求性能
- 实现难度：⭐⭐（中等）

## 推荐
对于独立开发者，推荐方案 1 或 5：
- 方案 1 功能完整，适合学习全栈开发
- 方案 5 性能好，维护成本低
```

### 功能发散：挖掘隐藏需求

当确定产品形态后，可以继续发散功能点：

```
你：我选择做传统博客。除了基本的文章发布，还有哪些功能可以考虑？

AI：按优先级分类：

## 核心功能（必须有）
- 文章 CRUD
- Markdown 编辑器
- 分类和标签

## 增强功能（建议有）
- 全文搜索
- 文章归档（按时间）
- RSS 订阅
- SEO 优化（sitemap、meta）

## 进阶功能（锦上添花）
- 评论系统（或集成 Giscus）
- 文章阅读统计
- 深色模式
- 多语言支持

## 高级功能（可选）
- 草稿自动保存
- 定时发布
- 文章协作编辑
- 会员订阅
```

### 技术方案发散

同一个功能，可能有多种实现方式：

```
你：文章编辑器应该怎么做？

AI：有 3 种主流方案：

## 方案 1：纯 Markdown
- 实现：textarea + marked.js 预览
- 优点：实现简单、存储方便
- 缺点：非技术用户学习成本高

## 方案 2：富文本编辑器
- 工具：TipTap、Quill、Editor.js
- 优点：所见即所得、用户友好
- 缺点：存储格式复杂、跨平台麻烦

## 方案 3：块编辑器（Notion 风格）
- 工具：Editor.js、Plate.js
- 优点：灵活、现代
- 缺点：实现复杂

## 推荐
初期用方案 1，快速上线后再迭代
```

### 发散后的收敛

头脑风暴的目的不是收集所有可能，而是**找到最适合当前阶段的方案**：

```mermaid
graph LR
    A["一个想法"] --> B["AI 发散\n5+ 种可能"]
    B --> C["你来筛选\n2-3 个候选"]
    C --> D["深入分析\n比较优劣"]
    D --> E["确定方案\n1 个最优解"]
```

### 实用建议

1. **先发散再收敛**：不要一开始就否定可能性
2. **让 AI 对比**：给出候选方案，让 AI 分析优劣
3. **考虑资源约束**：独立开发 vs 团队开发的方案不同
4. **分阶段实现**：选择可渐进式扩展的方案
