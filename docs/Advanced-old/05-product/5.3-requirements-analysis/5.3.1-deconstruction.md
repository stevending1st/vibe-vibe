# 5.3.1 如何看透题目要求——AI 辅助需求拆解

### 一句话破题

让 AI 帮你**从一句话需求中挖出十个具体问题**。

### 需求拆解的价值

一个看似简单的需求，背后往往藏着大量细节：

```
原始需求："做一个用户登录功能"

隐藏的问题：
- 用什么方式登录？邮箱？手机号？第三方？
- 密码有什么要求？长度？复杂度？
- 忘记密码怎么办？
- 需要验证码吗？什么情况下触发？
- 登录失败几次会锁定账号？
- Session 还是 JWT？有效期多久？
- 需要"记住我"功能吗？
```

### 让 AI 帮你提问

**Prompt 模板**：

```
我要实现 [功能描述]。

请扮演一个资深产品经理，从以下角度帮我列出需要澄清的问题：
1. 用户角度：谁会使用？使用场景是什么？
2. 功能角度：具体包含哪些子功能？
3. 边界角度：什么情况不支持？
4. 技术角度：有什么技术约束？
5. 验收角度：怎么判断做完了？
```

**示例对话**：

```
你：我要实现用户登录功能

AI（产品经理视角）：

## 需要澄清的问题

### 用户角度
- 目标用户是谁？普通用户还是企业用户？
- 用户主要在什么场景下登录？PC/移动端/都有？

### 功能角度
- 支持哪些登录方式？
  - [ ] 邮箱+密码
  - [ ] 手机号+验证码
  - [ ] 第三方登录（微信/Google/GitHub）
- 是否需要"记住登录状态"？
- 是否需要多设备登录限制？

### 边界角度
- 密码要求：最小长度？需要包含特殊字符？
- 登录失败处理：几次失败后锁定？锁定多久？
- 是否需要验证码？什么情况下触发？

### 技术角度
- 认证方式：Session 还是 JWT？
- Token 有效期多久？
- 是否需要 refresh token？

### 验收角度
- 正常登录成功后跳转到哪里？
- 登录失败要显示什么错误信息？
```

### 拆解后的输出

经过 AI 辅助拆解后，你应该能得到：

1. **确定的决策**：邮箱+密码登录，不支持第三方
2. **明确的规则**：密码 8-20 位，必须包含字母和数字
3. **清晰的边界**：失败 5 次锁定 30 分钟
4. **验收标准**：登录成功返回 token，失败返回错误码

### 实用建议

1. **先让 AI 提问，再自己回答**：AI 提的问题比你想的全面
2. **不确定的先记录**：暂时回答不了的问题标记为"待定"
3. **形成决策文档**：把问答整理成文档，作为后续开发的依据
