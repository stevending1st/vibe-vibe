# 5.5.2 我们要解决什么——问题陈述

### 一句话破题

问题陈述回答三个核心问题：**现状是什么、目标是什么、约束是什么**。

### 问题陈述的结构

```markdown
## 问题陈述

### 现状（As-Is）
当前用户面临什么问题？痛点是什么？

### 目标（To-Be）
理想状态是什么？解决后会怎样？

### 约束（Constraints）
有哪些限制条件？时间、资源、技术？
```

### 实际案例

```markdown
## 问题：技术博客内容难以管理

### 现状
- 技术人员使用第三方平台写博客，受制于平台审核
- 内容分散在多个平台，难以统一管理
- 担心平台关闭导致内容丢失
- 现有方案（如 Hexo）配置繁琐

### 目标
- 拥有完全可控的个人博客系统
- 支持 Markdown 写作，专注内容
- 良好的 SEO，被搜索引擎收录
- 部署简单，维护成本低

### 约束
- 独立开发者，开发时间有限
- 预算有限，尽量使用免费资源
- 技术栈限定：Next.js + PostgreSQL
- 两周内完成 MVP
```

### 好的问题陈述 vs 差的问题陈述

| 差的 | 好的 |
|------|------|
| "用户体验不好" | "用户平均需要点击 5 次才能完成注册，导致 60% 的用户放弃" |
| "性能需要优化" | "首页加载时间超过 5 秒，在 3G 网络下无法正常使用" |
| "功能不够完善" | "用户无法搜索历史文章，只能翻页浏览" |

### 让 AI 帮你完善问题陈述

```
你：我想做一个博客系统，帮我写问题陈述

AI：## 问题陈述：个人技术博客

### 现状分析
1. **平台依赖**
   - 技术文章发布在 CSDN/掘金等平台
   - 受制于平台审核，敏感内容可能被删
   - 广告干扰阅读体验

2. **内容管理困难**
   - 文章分散在多个平台
   - 缺乏统一的分类和标签体系
   - 无法控制内容的展示方式

3. **迁移风险**
   - 平台可能调整政策或关闭
   - 导出功能有限，迁移困难

### 目标状态
1. **完全可控**：自主托管，数据完全归属于自己
2. **专注写作**：支持 Markdown，无需关心排版
3. **可被发现**：良好的 SEO，被搜索引擎收录
4. **易于维护**：部署简单，无需频繁运维

### 约束条件
- 时间：2 周内完成 MVP
- 技术：使用 Next.js 16 + Prisma
- 成本：服务器费用控制在 $10/月以内
- 人力：一人开发
```


### 验证问题陈述

问自己几个问题：

1. **这是真问题吗？** - 是否有足够多的人遇到这个问题
2. **问题描述具体吗？** - 能否用数据量化
3. **目标可衡量吗？** - 如何判断问题已解决
4. **约束合理吗？** - 是否能在约束内达成目标

### 实用建议

1. **先问"为什么"**：挖掘问题背后的真正原因
2. **用数据说话**：尽量用数字描述现状
3. **明确优先级**：哪些问题必须解决，哪些可以接受
4. **考虑权衡**：有些问题的解决可能带来新问题
